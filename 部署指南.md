## 🚀 方案一：Railway 一键部署（推荐新手）

### 步骤
1. 点击 [![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new/template/PwpGcf)
2. 登录 Railway（可用 GitHub）
3. 点击 "Deploy Now"
4. 等待 3-5 分钟
5. 在 Railway Dashboard 找到你的网址
6. 在 Deploy Logs 中找到登录链接

### 优点
- ✅ 零配置，全自动
- ✅ 自动 HTTPS 证书
- ✅ 自动备份
- ✅ Web 界面管理

### 缺点
- ❌ 需要绑定信用卡
- ❌ 免费额度有限（$5 credit）
- ❌ 数据在美国（如果在意地域）

### 成本估算
```
Cap Web:     ~$10/月（CPU + 内存）
MySQL:       ~$5/月
MinIO:       ~$3/月
总计：       ~$18/月
```

## 🐳 方案二：Docker Compose 自己部署

### 前置要求
- 一台 Linux 服务器（1核2G 最低，推荐 2核4G）
- 已安装 Docker 和 Docker Compose
- 有基本的 Linux 命令行知识

### 完整部署步骤

#### 1. 准备服务器
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
sudo apt install docker-compose -y

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### 2. 下载配置文件
```bash
# 创建目录
mkdir ~/cap-deployment
cd ~/cap-deployment

# 下载 docker-compose 模板
wget https://raw.githubusercontent.com/CapSoftware/Cap/main/docker-compose.template.yml -O docker-compose.yml
```

#### 3. 修改配置文件
```bash
nano docker-compose.yml
```

**必须修改的地方：**
```yaml
environment:
  # 1. 改成你的域名或 IP
  WEB_URL: http://你的域名.com
  NEXTAUTH_URL: http://你的域名.com

  # 2. 生成随机密钥（超重要！）
  DATABASE_ENCRYPTION_KEY: 生成一个64位十六进制字符串
  NEXTAUTH_SECRET: 生成另一个64位十六进制字符串

  # 3. 修改存储密码
  CAP_AWS_ACCESS_KEY: 你的S3用户名
  CAP_AWS_SECRET_KEY: 你的S3密码

  # 4. 修改 MySQL 密码
  MYSQL_ROOT_PASSWORD: 你的数据库密码
```

**生成随机密钥的命令：**
```bash
# 生成密钥
openssl rand -hex 32
# 运行两次，分别用于 DATABASE_ENCRYPTION_KEY 和 NEXTAUTH_SECRET
```

#### 4. 启动服务
```bash
# 启动所有容器
docker-compose up -d

# 查看日志
docker-compose logs -f cap-web

# 等待看到 "Ready in XXXms" 表示启动成功
```

#### 5. 验证部署
```bash
# 检查所有容器是否运行
docker-compose ps

# 应该看到 3 个容器都是 "Up" 状态：
# - cap-web
# - cap-primary-db (MySQL)
# - cap-minio-storage
```

#### 6. 获取登录链接
```bash
# 查看 cap-web 日志，找登录魔法链接
docker-compose logs cap-web | grep "http://"
```

#### 7. 配置域名（可选但推荐）
```bash
# 安装 Nginx
sudo apt install nginx -y

# 创建反向代理配置
sudo nano /etc/nginx/sites-available/cap
```

Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name 你的域名.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/cap /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 8. 配置 HTTPS（强烈推荐）
```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 自动配置 SSL
sudo certbot --nginx -d 你的域名.com
```

## 📋 必需的环境变量详解

### 核心配置
| 变量 | 说明 | 示例 | 必填 |
|------|------|------|------|
| `DATABASE_URL` | MySQL 连接串 | `mysql://root:密码@localhost:3306/cap` | ✅ |
| `WEB_URL` | 你的网站地址 | `https://cap.yourdomain.com` | ✅ |
| `NEXTAUTH_URL` | 认证回调地址 | 同 WEB_URL | ✅ |
| `NEXTAUTH_SECRET` | 会话加密密钥 | 随机 64 位十六进制 | ✅ |
| `DATABASE_ENCRYPTION_KEY` | 数据库加密密钥 | 随机 64 位十六进制 | ✅ |

### S3 存储配置（必填）
| 变量 | 说明 | 示例 |
|------|------|------|
| `CAP_AWS_BUCKET` | 存储桶名称 | `cap-videos` |
| `CAP_AWS_REGION` | 区域 | `us-east-1` |
| `CAP_AWS_ACCESS_KEY` | 访问密钥 | `minioadmin` |
| `CAP_AWS_SECRET_KEY` | 密钥 | `minioadmin123` |
| `S3_PUBLIC_ENDPOINT` | 公开访问地址 | `http://你的域名:3902` |

### 可选配置（增强功能）
| 变量 | 功能 | 费用 |
|------|------|------|
| `RESEND_API_KEY` | 邮件登录 | 免费 100 封/天 |
| `GROQ_API_KEY` | AI 字幕/摘要 | 免费（有限额） |
| `OPENAI_API_KEY` | AI 备用方案 | 按量付费 |
| `STRIPE_SECRET_KEY` | 收费功能 | 免费 + 手续费 |
| `POSTHOG_API_KEY` | 用户分析 | 免费 1M 事件/月 |

## ⚠️ 常见问题

### 1. 容器启动失败
```bash
# 查看详细错误
docker-compose logs cap-web

# 常见原因：
# - 端口被占用（3000, 3306, 3902）
# - 环境变量配置错误
# - 磁盘空间不足
```

### 2. 无法连接数据库
```bash
# 检查 MySQL 容器
docker-compose exec ps-mysql mysql -u root -p

# 手动创建数据库
CREATE DATABASE IF NOT EXISTS planetscale;
```

### 3. 桌面应用无法上传
- 检查 `WEB_URL` 是否正确
- 确保防火墙开放 3000 端口
- 桌面应用设置中填写正确的服务器地址

### 4. 视频无法播放
- 检查 MinIO 是否正常运行
- 确认 `S3_PUBLIC_ENDPOINT` 可以公网访问
- 检查存储桶权限
