## 准备工作

### 服务器要求
```
最低配置：2核4G + 50GB 磁盘
推荐配置：4核8G + 100GB+ 磁盘
操作系统：Ubuntu 20.04/22.04 或 CentOS 7/8
```

### 必需软件
```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

## 方案选择

### 方案对比

| 维度 | MinIO（自建） | 阿里云 OSS | 腾讯云 COS |
|------|--------------|-----------|-----------|
| **成本** | 免费（占服务器磁盘） | ¥0.12/GB/月 | ¥0.118/GB/月 |
| **速度** | 快（本地读写） | 很快（CDN） | 很快（CDN） |
| **带宽** | 受限于服务器带宽 | 可扩展 | 可扩展 |
| **可靠性** | 需要自己备份 | 99.9999999% | 99.9999999% |
| **适合场景** | 小团队、初期 | 中大规模 | 中大规模 |

### 推荐策略
```
启动阶段：MinIO（0成本）
  ↓ 视频量 > 100GB
迁移到云存储：阿里云/腾讯云
  ↓ 同时配置
CDN 加速：提升用户体验
```

## 快速部署（MinIO）

### 第一步：下载配置文件
```bash
# 创建部署目录
mkdir -p ~/cap-deployment
cd ~/cap-deployment

# 使用国内优化版配置
wget https://raw.githubusercontent.com/你的仓库/Cap/main/docker-compose-china.yml -O docker-compose.yml

# 或者手动创建（如果网络不好）
nano docker-compose.yml
# 复制 docker-compose-china.yml 的内容
```

### 第二步：生成安全密钥
```bash
# 生成两个随机密钥
echo "DATABASE_ENCRYPTION_KEY:"
openssl rand -hex 32

echo "NEXTAUTH_SECRET:"
openssl rand -hex 32

# 记下这两个值，填入 docker-compose.yml
```

### 第三步：修改配置文件
```bash
nano docker-compose.yml
```

**必须修改的 5 个地方：**
```yaml
1. WEB_URL: https://你的域名.com
2. NEXTAUTH_URL: https://你的域名.com
3. DATABASE_ENCRYPTION_KEY: [刚才生成的第一个密钥]
4. NEXTAUTH_SECRET: [刚才生成的第二个密钥]
5. MYSQL_ROOT_PASSWORD: [设置一个强密码]
6. S3_PUBLIC_ENDPOINT: https://你的域名.com:9000
7. MINIO_ROOT_PASSWORD: [设置 MinIO 密码]
```

### 第四步：启动服务
```bash
# 拉取镜像（首次较慢）
docker-compose pull

# 启动所有服务
docker-compose up -d

# 查看启动状态
docker-compose ps

# 查看日志
docker-compose logs -f cap-web
```

### 第五步：配置 MinIO
```bash
# 1. 访问 MinIO 控制台
http://你的服务器IP:9001

# 2. 登录
用户名: cap-admin
密码: [你在 docker-compose.yml 中设置的 MINIO_ROOT_PASSWORD]

# 3. 创建存储桶
- 点击 "Buckets" → "Create Bucket"
- 名称: cap-videos（必须和配置文件中的 CAP_AWS_BUCKET 一致）
- 点击创建

# 4. 设置存储桶为公开读取
- 进入 cap-videos 存储桶
- 点击 "Access" → "Access Policy"
- 设置为 "Public"（允许匿名读取）
```

**或者用命令行配置：**
```bash
# 进入 MinIO 容器
docker exec -it cap-minio sh

# 安装 mc 客户端（如果没有）
mc alias set myminio http://localhost:9000 cap-admin cap-admin-password-2024

# 创建存储桶
mc mb myminio/cap-videos

# 设置公开访问策略
mc anonymous set download myminio/cap-videos

# 退出容器
exit
```

### 第六步：配置 Nginx 反向代理
```bash
# 安装 Nginx
sudo apt install nginx -y

# 创建配置文件
sudo nano /etc/nginx/sites-available/cap
```

**Nginx 配置：**
```nginx
# Cap Web 主站
server {
    listen 80;
    server_name 你的域名.com;

    # 限制上传大小（重要！视频文件可能很大）
    client_max_body_size 2G;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置（上传大文件需要）
        proxy_connect_timeout 600;
        proxy_send_timeout 600;
        proxy_read_timeout 600;
    }
}

# MinIO 存储（视频访问）
server {
    listen 80;
    server_name storage.你的域名.com;  # 或者用主域名 + 9000 端口

    # 限制上传大小
    client_max_body_size 2G;

    location / {
        proxy_pass http://localhost:9000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # MinIO 需要的特殊头
        proxy_set_header Connection "";
        chunked_transfer_encoding off;

        # 超时设置
        proxy_connect_timeout 300;
        proxy_send_timeout 300;
        proxy_read_timeout 300;
    }
}
```

启用配置：
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/cap /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 第七步：配置 HTTPS（强烈推荐）
```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 自动配置 SSL（为两个域名）
sudo certbot --nginx -d 你的域名.com -d storage.你的域名.com

# 自动续期（已自动配置，检查一下）
sudo systemctl status certbot.timer
```

### 第八步：验证部署
```bash
# 1. 检查所有容器状态
docker-compose ps
# 应该看到 3 个容器都是 "Up"

# 2. 检查 Cap Web 日志，找登录链接
docker-compose logs cap-web | grep "http"

# 3. 访问网站
https://你的域名.com

# 4. 测试视频上传（通过桌面应用）
# 在桌面应用设置中填入：https://你的域名.com
```

## 云存储方案

### 阿里云 OSS

#### 1. 创建 OSS 存储桶
```bash
1. 登录阿里云控制台
2. 进入 OSS 产品页面
3. 创建 Bucket：
   - 名称: cap-videos-prod
   - 区域: 选离你服务器最近的（如华北2北京）
   - 读写权限: 公共读
   - 服务器端加密: 不加密（或按需）
```

#### 2. 获取访问密钥
```bash
1. 右上角头像 → AccessKey 管理
2. 创建 AccessKey
3. 记录：
   - AccessKey ID
   - AccessKey Secret
```

#### 3. 修改 docker-compose.yml
```yaml
cap-web:
  environment:
    # 删除 MinIO 相关配置，改用阿里云 OSS
    CAP_AWS_ACCESS_KEY: LTAI5t... # 你的 AccessKey ID
    CAP_AWS_SECRET_KEY: xxxxx     # 你的 AccessKey Secret
    CAP_AWS_BUCKET: cap-videos-prod
    CAP_AWS_REGION: oss-cn-beijing  # 改成你的区域
    CAP_AWS_ENDPOINT: https://oss-cn-beijing.aliyuncs.com  # 阿里云 OSS endpoint

    # 公开访问地址
    S3_PUBLIC_ENDPOINT: https://cap-videos-prod.oss-cn-beijing.aliyuncs.com
    # 如果配置了自定义域名或 CDN：
    # CAP_AWS_BUCKET_URL: https://cdn.你的域名.com
```

#### 4. 重启服务
```bash
# 停止 MinIO（不再需要）
docker-compose stop minio

# 重启 Cap Web
docker-compose restart cap-web

# 查看日志
docker-compose logs -f cap-web
```

### 腾讯云 COS

配置类似阿里云，区别：
```yaml
CAP_AWS_REGION: ap-beijing  # 腾讯云区域格式
CAP_AWS_ENDPOINT: https://cos.ap-beijing.myqcloud.com
S3_PUBLIC_ENDPOINT: https://cap-videos-123456.cos.ap-beijing.myqcloud.com
```

### 七牛云

```yaml
# 七牛云也支持 S3 协议
CAP_AWS_REGION: cn-east-1
CAP_AWS_ENDPOINT: https://s3-cn-east-1.qiniucs.com
S3_PUBLIC_ENDPOINT: https://你的CDN域名.qiniup.com
```

## 性能优化

### 1. CDN 加速（推荐）

**为什么需要 CDN？**
- 视频文件大，直接从源站下载慢
- 减轻服务器带宽压力
- 提升用户体验

**配置步骤（以阿里云为例）：**
```bash
1. 阿里云控制台 → CDN
2. 添加域名：cdn.你的域名.com
3. 源站类型：OSS 域名
4. 源站地址：cap-videos-prod.oss-cn-beijing.aliyuncs.com
5. 开启 HTTPS
6. 配置完成后，修改 docker-compose.yml：

environment:
  CAP_AWS_BUCKET_URL: https://cdn.你的域名.com
```

### 2. 数据库优化

```yaml
# docker-compose.yml 中添加 MySQL 性能参数
ps-mysql:
  command:
    - --max_connections=1000
    - --innodb_buffer_pool_size=1G      # 改成服务器内存的 50-70%
    - --innodb_log_file_size=256M
    - --innodb_flush_log_at_trx_commit=2
    - --query_cache_type=1
    - --query_cache_size=64M
```

### 3. 限流配置（防止滥用）

在 Nginx 中添加：
```nginx
# 限制上传速率（防止恶意大文件上传）
limit_req_zone $binary_remote_addr zone=upload_limit:10m rate=10r/m;

server {
    location /api/upload {
        limit_req zone=upload_limit burst=5;
        proxy_pass http://localhost:3000;
    }
}
```

### 4. 日志轮转

```bash
# 创建 Docker 日志配置
sudo nano /etc/docker/daemon.json
```

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

```bash
sudo systemctl restart docker
```

## 监控与备份

### 1. 自动备份脚本

```bash
# 创建备份脚本
nano ~/cap-deployment/backup.sh
```

```bash
#!/bin/bash
BACKUP_DIR="/root/cap-backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份 MySQL
docker exec cap-mysql mysqldump -u root -pYourStrongPassword cap > $BACKUP_DIR/cap_db_$DATE.sql

# 备份 MinIO 数据（如果使用）
tar -czf $BACKUP_DIR/minio_data_$DATE.tar.gz -C /var/lib/docker/volumes/cap-deployment_minio-data/_data .

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $DATE"
```

```bash
# 添加执行权限
chmod +x ~/cap-deployment/backup.sh

# 设置定时任务（每天凌晨 2 点）
crontab -e
# 添加：
0 2 * * * /root/cap-deployment/backup.sh >> /var/log/cap-backup.log 2>&1
```

## 故障排查

### 问题 1：视频无法上传
```bash
# 检查 MinIO 是否正常
docker logs cap-minio

# 检查存储桶是否存在
docker exec -it cap-minio sh
mc ls myminio/
# 应该看到 cap-videos

# 检查网络连接
curl http://localhost:9000/minio/health/live
```

### 问题 2：视频无法播放
```bash
# 检查 S3_PUBLIC_ENDPOINT 是否可访问
curl https://你的域名.com:9000

# 检查 Nginx 配置
sudo nginx -t

# 检查防火墙
sudo ufw allow 9000
sudo ufw status
```

### 问题 3：数据库连接失败
```bash
# 进入 MySQL 容器测试
docker exec -it cap-mysql mysql -u root -pYourStrongPassword

# 检查数据库是否创建
SHOW DATABASES;
# 应该看到 'cap'

# 如果没有，手动创建
CREATE DATABASE cap CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 问题 4：内存不足
```bash
# 查看内存使用
docker stats

# 限制容器内存
docker-compose.yml 中添加：
services:
  cap-web:
    mem_limit: 2g
  ps-mysql:
    mem_limit: 1g
  minio:
    mem_limit: 512m
```

### 问题 5：磁盘空间不足
```bash
# 查看磁盘使用
df -h

# 清理 Docker 无用数据
docker system prune -a --volumes

# 查看各容器占用空间
docker system df -v
```

## 成本估算（国内）

### 小团队方案（MinIO）
```
服务器: Hetzner/Vultr €4.5-10/月
域名: ¥50/年

```
