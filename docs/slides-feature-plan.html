核心数据类型（TypeScript）
type SlideElement = {
  id: string;
  type: "title" | "bullet" | "shape" | "image";
  x: number; y: number;
  width: number; height: number;
  text?: string;
  roughness: number; // 0–3，手绘程度
  strokeColor: string;
  backgroundColor: string;
};

type SlideData = {
  id: string;
  index: number;
  title: string;
  elements: SlideElement[];
  speakerNotes: string; // 对应脚本段落
  recordingTimestamp?: number; // 录制时翻页时间点
};

type PresentationData = {
  slides: SlideData[];
  theme: "sketch" | "minimal" | "dark";
  aspectRatio: "16:9" | "4:3";
  script: string; // 原始脚本
};
关键技术决策
幻灯片渲染用什么库？
roughjs — 轻量级（20KB），专为手绘风格设计，Excalidraw 内部也用它。在 Canvas 上直接绘制，无需引入完整 Excalidraw。
AI 生成在哪里调用？
桌面 App 调用 Web API Route (`/api/slides/generate`)，在服务端用 Groq 生成。符合架构规范：AI 调用不在客户端。
录制时幻灯片如何集成？
新增 Presentation 录制模式，幻灯片渲染在独立的 Tauri 窗口（全屏），摄像头悬浮 overlay，截屏模式捕获幻灯片窗口内容。
幻灯片数据存在哪里？
本地：嵌入 .cap 项目文件（ProjectConfiguration 新增字段）。上传后：S3 路径 {userId}/{videoId}/slides.json。
是否需要编辑幻灯片？
第一期只做 AI 生成 + 预览 + 简单文字编辑（点击元素改文字）。完整拖拽编辑放第二期，避免过度设计。
建议开发阶段
阶段 1
基础幻灯片
  • SlideData 类型定义
  • roughjs 渲染器
  • 脚本输入 UI
  • AI 生成 API Route
  • 本地保存到 .cap
阶段 2
录制集成
  • Presentation 录制模式
  • 全屏幻灯片窗口
  • 翻页快捷键
  • 记录翻页时间戳
  • 摄像头 overlay
阶段 3
分享 & 导出
  • 上传 slides.json 到 S3
  • Web 分享页 SlidesViewer
  • 视频时间 ↔ 幻灯片同步
  • 导出 .excalidraw 文件
阶段 4
编辑增强
  • 元素拖拽编辑
  • 增删幻灯片页
  • 主题切换
  • 编辑器幻灯片轨道
确认后开始：请告知从哪个阶段开始，以及是否有调整。建议从 阶段 1 的类型定义 + roughjs 渲染器 + 脚本输入 UI 开始，建立可见的原型后再推进录制集成。
